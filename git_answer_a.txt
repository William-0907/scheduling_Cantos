My Best Practice in Using Git:

1. upon creating a project folder, initialize a git repository using the command "git init"
2. after initializing, in my case, cause i upload on GitHub a lot, I will create a remote repository on GitHub then connect my local repository immediately using the command "git remote add origin http://github.com/William-0907/My_git_repo.git"
3. Then i will make a file called ".gitignore" to put files and folders like my virtualenvironment and .env where i put my api keys.
4. Then ill proceed to do my project, after that, If I think Im done, I will then use "git add ." to stage my folder then ill commit to save it, i use commit messages that are easy to understand and remember.
5. if im planning to do a new part i will mnake a new branch using the command "git checkout -b <branch_name>", then here i will do the project, then if its works then i will again add and commit.
